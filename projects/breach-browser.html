Breach Browser

Python Elasticsearch Flask Docker Full-Text Search Web Interface

Project Overview

Breach Browser is a fast and efficient full-text search engine designed for large text datasets with a clean web interface. Built with Python, Elasticsearch, and Flask, it provides researchers and analysts with powerful search capabilities across massive document collections.

The application excels at processing and indexing large volumes of text data, making it ideal for investigating data breaches, analyzing document dumps, or searching through extensive text archives. It features advanced query syntax, progressive loading, and efficient chunking mechanisms.

Core Features

  • Full-Text Search

    Advanced query syntax supporting Boolean operations, phrase matching, wildcard searches, and fuzzy matching for comprehensive text analysis.

  • Efficient Chunking

    Intelligent processing of large files with configurable chunk sizes for optimal memory usage and search performance.

  • Progressive Loading

    Faster search results through progressive loading and efficient indexing with resume capabilities for interrupted operations.

  • Detailed Results

    Comprehensive search results with file metadata, context highlighting, and source attribution for accurate analysis.

  • Real-time Statistics

    Live indexing progress tracking and search statistics for monitoring performance and data processing status.

  • Resume-Capable Indexing

    Robust indexing system that can resume interrupted processes, preventing data loss and optimizing processing time.

Search Capabilities

Advanced Query Syntax

  • Boolean Operations: Use AND, OR, NOT operators for complex search logic
  • Phrase Matching: Exact phrase searches with quotation marks
  • Wildcard Search: Pattern matching with asterisk (*) for partial terms
  • Fuzzy Search: Similarity matching with tilde (~) for typo tolerance
  • Field-Specific: Target specific document fields for precise searches
  • Range Queries: Date and numeric range searching capabilities

Search Examples

  • term1 AND term2 - Find documents containing both terms
  • "exact phrase" - Find exact phrase matches
  • test* - Find words starting with "test"
  • term~ - Find similar terms (fuzzy search)
  • term1 OR term2 - Find documents with either term
  • NOT term - Exclude documents with this term

Technical Architecture

Backend Components

  • Elasticsearch Engine: High-performance search and analytics engine for full-text indexing
  • Flask Web Server: Lightweight Python web framework handling HTTP requests and responses
  • Document Processor: Intelligent text chunking and metadata extraction system
  • Indexing Manager: Batch processing with progress tracking and resume capabilities
  • Query Parser: Advanced query syntax processing and optimization

Performance Features

  • Chunked Processing: 500KB default chunk size for optimal memory usage
  • Batch Indexing: Configurable batch sizes (default: 5 documents) for efficient processing
  • Memory Management: Optimized for large datasets requiring substantial RAM
  • Concurrent Operations: Multi-threaded processing for improved performance

Installation & Setup

System Requirements

  • Python 3.8+ for application runtime
  • Docker for Elasticsearch container management
  • Sufficient storage space for datasets and search indices
  • Substantial RAM for large dataset processing (2GB+ recommended)

Docker Configuration

  • Elasticsearch 7.17.9 with optimized Java heap settings
  • Persistent data volumes for index storage
  • Configured memory limits and file descriptor settings
  • Security disabled for local development (configurable for production)

Configuration Options

Customizable Parameters

  • Chunk Size: Text processing chunk size (default: 500KB)
  • Batch Size: Number of documents per indexing batch (default: 5)
  • Port Configuration: Web interface port (default: 5000)
  • Index Name: Elasticsearch index identifier (default: text_documents)
  • Data Directory: Source file location (default: ./data)

Performance Tuning

  • Adjustable chunk sizes for different file types and sizes
  • Configurable batch processing for optimal throughput
  • Memory allocation settings for Elasticsearch optimization
  • Index refresh intervals for real-time vs. batch processing

Development Challenges & Solutions

Large Dataset Processing

Challenge: Efficiently processing and indexing massive text datasets without overwhelming system resources.

Solution: Implemented intelligent chunking with configurable sizes and batch processing with progress tracking to handle large files systematically.

Search Performance Optimization

Challenge: Delivering fast search results across large indexed datasets while maintaining relevance and accuracy.

Solution: Leveraged Elasticsearch's advanced indexing and query optimization features with custom tuning for text-heavy datasets.

Resumable Operations

Challenge: Handling interrupted indexing processes for large datasets that may take hours to complete.

Solution: Built resume-capable indexing with progress persistence, allowing operations to continue from interruption points.

Security Considerations

Local Development

  • No external network exposure in standard configuration
  • Local file system access only within designated data directories

Production Deployment

  • Restrict network access and implement firewall rules
  • Regular security updates and monitoring
