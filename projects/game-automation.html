Project Overview

Developed a comprehensive AI system that automates gameplay in Trackmania using computer vision and deep learning. The project combines real-time screen analysis, machine learning prediction, and automated control inputs to create an AI capable of playing the game autonomously.

Key Features

  • Real-time Screen Analysis

    Utilizes OpenCV for processing game screen captures, including edge detection and line recognition for understanding the track layout.

  • Machine Learning Model

    Implements a CNN using TensorFlow for predicting optimal driving actions based on screen state.

  • Automated Control System

    Uses PyDirectInput for precise game control, simulating keyboard inputs based on AI predictions.

  • Data Collection Pipeline

    Custom data gathering system for recording gameplay and corresponding control inputs for model training.

Technical Implementation

Screen Capture and Processing

The system captures a specific region of the game screen using MSS for efficiency. The captured images are processed using OpenCV with the following pipeline:

  • Conversion to grayscale
  • Gaussian blur application
  • Canny edge detection
  • HoughLinesP algorithm for line detection

Data Collection

Created a sophisticated data collection system that:

  • Captures screen states at regular intervals
  • Records corresponding keyboard inputs
  • Saves data in a structured format for training
  • Implements CSV logging for maintaining data relationships

Model Architecture

Developed a CNN with the following structure:

  • Input layer accepting 224x224x3 images
  • Multiple convolutional and max pooling layers
  • Dense layers for feature interpretation
  • Output layer for three-class prediction (up, up&left, up&right)

Challenges & Solutions

Timing Precision

Challenge: Achieving precise timing for key inputs when simulating analog controls with digital inputs.

Solution: Implemented fine-tuned timing system for key press durations, allowing for more nuanced control.

Input Simulation

Challenge: Initial issues with pyautogui not working effectively in the game environment.

Solution: Switched to PyDirectInput, which provided more reliable and consistent input simulation.
